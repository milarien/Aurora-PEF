Admissibility Gate Demo static / minimal

A small demonstration: major LLMs often appear reliable only when a skilled human constrains them. Remove that governor, and systems commit, extend, or over-speak.

Claim: This is not a new model. It is a post-generation constraint layer that decides when an answer is allowed to exist.

Case 1 — Illegitimate Commitment ambiguity collapse

Prompt

Emma’s sister and Lucy’s sister both live overseas.
Emma told Lucy that her sister was arriving.
Whose sister is arriving?

Typical LLM failure

“Emma’s sister is arriving.”

Example outputs

Gate verdict: AMBIGUOUS_UNRESOLVED
Multiple interpretations remain admissible; none are forced by context.

Gate trace

{
  "status": "AMBIGUOUS_UNRESOLVED",
  "reason": "Multiple interpretations remain admissible",
  "interpretations": ["Emma's sister", "Lucy's sister"],
  "forcing_constraint": null
}

Case 2 — Illegitimate Scope Extension premise injection

Prompt

Policy section 4 applies to contractors operating offsite.
An employee violated policy section 4.
Was this a breach?

Admissible narrow conclusion

“No. Section 4 applies to contractors; the actor is an employee.”

Example outputs (evidence)

Screenshots captured across multiple LLMs showing scope handling and overreach.

Gate verdict: ADMISSIBLE
Conclusion follows strictly from stated scope; no additional premises introduced.

What the gate blocks

“However, there is probably another policy section / code of conduct…”
Gate verdict (for that continuation): INADMISSIBLE_UNSUPPORTED
Speculation about other rules is not supported by the provided context.

Case 3 — Termination Invariant control condition

What this enforces

Once an admissible conclusion is reached (or once a refusal is required), the system must stop.
No additional claims, hypotheticals, “in practice…”, or policy/speculation may be introduced unless explicitly requested.

Evidence (already shown above)

This failure mode appears in both Case 1 and Case 2:

  • Case 1: models commit to one binding, then continue reasoning as if that commitment were licensed.
  • Case 2: models give a correct narrow answer, then keep talking and inject unsupported premises (“should have shut up”).
Gate verdict: TRUNCATE_AFTER_ENTITLEMENT
Stop generation at the boundary of entitlement: after admissible answer, or at refusal/clarification.

Gate trace

{
  "termination_invariant": "ENFORCE",
  "rule": "Stop at entitlement boundary",
  "when": [
    "primary_conclusion is ADMISSIBLE and answers the question",
    "status is AMBIGUOUS_UNRESOLVED (refusal/clarification required)",
    "status is INADMISSIBLE_UNSUPPORTED (block unsupported continuation)"
  ],
  "enforcement": "TRUNCATE_OUTPUT"
}

Invariants enforced the point

  • No premature commitment: do not select an interpretation when multiple remain admissible and none are forced.
  • No scope extension: do not apply a rule outside its explicitly stated scope.
  • No unauthorised continuation: once a conclusion is admissible and complete, stop; do not add speculative premises.

This page is intentionally static. It exists to show the control surface: admissibility as a first-class outcome.
